# Figure 17: Framework Validation 说明文档

## 图表概述

本图以散点图形式展示了评估框架预测得分与应用层实际性能之间的相关关系。横轴为框架预测的综合得分，纵轴为应用性能指标，每种协议以不同颜色和标记的散点表示，误差棒反映测量不确定性，红色虚线表示线性回归拟合结果，浅红色区域表示置信区间。

## 验证方法论

框架验证旨在检验综合评估得分对实际应用性能的预测能力。验证采用安全线性回归作为基准应用场景，该应用涉及向量内积计算，能够充分利用OLE原语的批量计算特性。

应用性能指标定义为执行时间的倒数乘以缩放系数，数值越大表示性能越优。该定义确保性能指标的方向与综合得分的方向一致，便于相关性分析。

Pearson相关系数 $r$ 用于量化预测得分与实际性能之间的线性相关程度。$r$ 值接近1表示强正相关，接近0表示无相关，接近-1表示强负相关。图中左上角标注了计算得到的相关系数值。

## 验证结果解读

散点分布的整体趋势反映了得分与性能之间的关系模式。若散点沿回归线紧密分布，表明框架预测与实际性能吻合程度较高。若散点偏离回归线较远，表明存在框架未能捕捉的性能影响因素。

置信区间的宽度反映了预测的不确定性。区间较窄表明预测可信度较高，区间较宽则表明预测存在较大不确定性，可能需要更多数据或改进模型。

误差棒的大小反映了各协议性能测量的变异程度。误差棒较大的协议其实际性能波动较大，可能受实现细节或运行环境影响较为突出。
