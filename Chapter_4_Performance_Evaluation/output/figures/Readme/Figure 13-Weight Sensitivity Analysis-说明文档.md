# Figure 13: Weight Sensitivity Analysis 说明文档

## 图表概述

本图展示了五种OLE协议的综合得分在不同权重配置下的变化情况。横轴为五种权重配置方案，纵轴为综合得分，每种协议以折线和散点表示，折线斜率和交叉情况反映了协议对权重变化的敏感程度。

## 权重配置方案

基准配置将通信效率权重设为0.35，轮复杂度权重设为0.20，计算效率权重设为0.25，安全裕度权重设为0.20。该配置反映了通信带宽通常是分布式安全计算的主要瓶颈这一经验认知。

通信优先配置将通信效率权重提升至0.50，适用于带宽受限的网络环境如移动网络或跨洲际通信场景。

计算优先配置将计算效率权重提升至0.45，适用于计算资源受限的场景如嵌入式设备或实时计算应用。

均衡配置对四个维度赋予相等的0.25权重，用于无明确优先级偏好的通用场景。

安全优先配置将安全裕度权重提升至0.40，适用于对安全性要求较高的金融或政务应用场景。

## 敏感性分析结果

折线的斜率反映协议对特定权重变化的敏感程度。斜率陡峭表明该协议的排名容易受权重调整的影响，斜率平缓则表明排名较为稳健。

折线之间的交叉点标识了协议排名发生逆转的权重阈值。交叉点越多，表明协议间的相对优劣关系对权重配置越敏感。

部分协议在所有权重配置下均保持较高或较低得分，表明这些协议的综合表现相对稳定，不易因评估偏好的调整而改变其相对位置。
