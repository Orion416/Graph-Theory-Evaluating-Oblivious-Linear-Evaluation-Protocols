# Figure 16: Ranking Inconsistency 说明文档

## 图表概述

本图采用冲积图形式展示五种OLE协议在不同评估方法下的排名变化轨迹。横轴为四种评估方法，纵轴为排名位次，每种协议以不同颜色的流带表示，流带的交叉和起伏反映了排名的不一致性。

## 评估方法说明

朴素通信排名仅依据通信复杂度对协议进行排序，通信量越低排名越靠前。该方法忽略了计算开销、轮复杂度和安全性等其他维度。

朴素计算排名仅依据计算复杂度进行排序，计算开销越低排名越靠前。该方法同样忽略了其他评估维度。

朴素组合排名采用通信量与计算量的简单加权求和进行排序，但权重设定缺乏系统性依据，且未考虑轮复杂度和安全裕度。

框架排名采用本文提出的多维综合评估框架，综合考虑通信效率、轮复杂度、计算效率和安全裕度四个维度，权重配置基于应用场景特征设定。

## 排名不一致性分析

流带的交叉表示协议之间的相对排名发生了逆转。交叉越频繁，表明不同评估方法对协议优劣的判定分歧越大。

部分协议的流带在多个方法间保持平行，表明这些协议在各单一维度上的相对表现一致，其排名对评估方法不敏感。

部分协议的流带出现剧烈起伏，从高排名下降至低排名或反之。这类协议在不同维度上的表现差异较大，评估方法的选择对其排名影响较为突出。

从朴素方法到框架方法的过渡中，排名变化反映了多维综合评估相对于单维度评估的差异。综合评估能够识别那些在多个维度上表现均衡的协议。

## 与论文内容的对应关系

本图对应论文中排名一致性分析部分，说明了单维度评估方法的局限性以及多维综合评估框架的必要性。
