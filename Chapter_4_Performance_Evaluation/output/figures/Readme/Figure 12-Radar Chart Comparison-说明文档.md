# Figure 12: Radar Chart Comparison 说明文档

## 图表概述

本图采用雷达图形式对五种OLE协议进行多维性能对比。四个坐标轴分别对应通信效率、轮复杂度、计算效率和安全裕度四个评估维度，每种协议以封闭多边形表示，多边形面积越大代表综合性能越优。

## 评估维度说明

通信效率维度采用归一化后的通信复杂度倒数计算，通信量越低则该维度得分越高。归一化过程采用对数尺度的最小-最大变换，计算公式为 $S_{comm} = 1 - \frac{\log C - \log C_{min}}{\log C_{max} - \log C_{min}}$，其中 $C$ 为通信量。

轮复杂度维度直接采用交互轮数的归一化倒数，轮数越少得分越高。计算公式为 $S_{round} = 1 - \frac{R - R_{min}}{R_{max} - R_{min}}$，其中 $R$ 为总轮数。

计算效率维度采用与通信效率相同的归一化方法，计算开销越低得分越高。

安全裕度维度反映协议的安全性强度，基于密码学假设的保守程度和UC安全性等因素确定。该维度直接采用线性归一化，数值越高表示安全性越强。

## 各协议的多维特征

RLWE-OLE在通信效率和安全裕度两个维度表现较好，但计算效率维度相对较弱，整体多边形呈现向左上方倾斜的形态。

IKNP-OLE在计算效率维度占优，但通信效率维度表现不佳，多边形向右下方倾斜。

PCG-OLE的多边形较为均衡，四个维度得分相对接近，面积处于中等水平。

HE-OLE在通信效率维度表现较好，但计算效率和轮复杂度维度较弱，整体面积较小。

Noisy-Encoding OLE的安全裕度维度因UC安全性获得加成，但其他维度表现一般。
