# Figure 9: Parameter Space Mapping 说明文档

## 图表概述

本图采用三维散点图的形式展示了五种OLE协议在密码学参数空间中的分布特征。三个坐标轴分别对应域大小、安全级别和问题规模，每种协议以不同颜色和标记符号表示，散点大小随问题规模变化。

## 参数空间定义

横轴表示域大小的对数值 $\log_2(|F|)$，其中 $|F|$ 为协议运算所在有限域的阶。对于基于环的协议如RLWE-OLE和HE-OLE，域大小由环维度 $n$ 与模数位长 $\log_2 q$ 的乘积确定。对于基于OT扩展的协议如IKNP-OLE，域大小直接等于域元素的位长度。

纵轴表示安全级别，以比特为单位。本实验中所有协议均配置为128比特计算安全性，这是当前密码学标准推荐的安全强度。

深度轴表示问题规模的对数值 $\log_2 N$，其中 $N$ 为OLE实例数量。实验覆盖了从 $2^{10}$ 到 $2^{20}$ 的规模范围，对应实际安全计算应用中常见的工作负载。

## 协议分布特征

RLWE-OLE和HE-OLE位于参数空间的高域大小区域，这反映了基于格的协议需要较大的环维度和模数以保证安全性与正确性。IKNP-OLE、Noisy-Encoding OLE和PCG-OLE则聚集在较低域大小区域，表明基于OT扩展的协议能够在较小的域上高效运作。

各协议在问题规模维度上均呈现线性延伸特征，表明它们都能够处理不同规模的OLE计算任务。连接各规模点的虚线辅助展示了单一协议随规模变化的轨迹。
