# Figure 11: Computation-Communication Trade-off 说明文档

## 图表概述

本图以双对数散点图形式展示了五种OLE协议在计算开销与通信开销二维空间中的分布。横轴为通信量，纵轴为计算量，每种协议的多个规模点以渐变色标识，较大的点对应较大的问题规模。红色虚线标注了Pareto前沿。

## 权衡关系的理论背景

安全计算协议的设计通常面临计算与通信之间的权衡。降低通信复杂度往往需要更多的本地计算作为代价，反之亦然。这种权衡关系源于密码学原语的固有特性，例如同态加密能够减少交互轮数和通信量，但引入了高昂的密文运算开销。

Pareto前沿由那些在两个维度上均无法被其他方案同时超越的点构成。位于前沿上的协议代表了当前技术水平下计算-通信权衡的最优折中方案。

## 各协议的权衡特征

RLWE-OLE和HE-OLE位于图的左上区域，表现为低通信但高计算的特征。这反映了基于格的协议需要执行多项式环上的NTT变换和模运算，计算开销较大，但打包技术使得通信效率较高。

IKNP-OLE位于图的右下区域，呈现高通信但低计算的特征。OT扩展主要依赖哈希运算，计算效率较高，但每个OLE需要传输与域大小成正比的数据量。

PCG-OLE在中等规模下接近Pareto前沿，在通信和计算两个维度上均保持适中水平。其亚线性通信优势在大规模场景下更为突出。

Noisy-Encoding OLE的位置介于IKNP-OLE和基于格的协议之间，多项式编码增加了计算负担但未能有效降低通信量。
