# Figure 8: Experimental Network Topology 说明文档

## 图表概述

本图展示了OLE协议性能评估框架所采用的实验网络拓扑结构。该拓扑由两个计算参与方、一个网络模拟器、一个控制服务器以及分布式测量探针组成，用于支撑双方安全计算协议的性能测量与分析。

## 拓扑结构描述

实验网络采用对称式双方计算架构。参与方 $P_1$ 与参与方 $P_2$ 分别部署于独立的计算节点上，每个节点配置相同的硬件资源以确保测量结果的公平性。两个参与方通过中央网络模拟器进行通信，该模拟器能够模拟从局域网到广域网的多种网络条件，带宽范围覆盖10 Mbps至1 Gbps，延迟范围覆盖0.1毫秒至100毫秒。

控制服务器位于网络拓扑的下层，负责协调实验流程、同步参与方状态以及收集运行日志。该服务器与网络模拟器及两个参与方均建立控制通道连接，但不参与协议的实际数据传输，从而避免对测量结果产生干扰。

测量探针分别部署于两个参与方节点上，用于采集通信量、计算时间、内存占用等性能指标。探针采用非侵入式设计，通过系统级接口获取资源使用数据，对协议执行的性能影响可忽略不计。

## 网络模拟机制

网络模拟器支持可配置的带宽限制和延迟注入。带宽控制通过令牌桶算法实现，能够精确模拟不同网络环境下的吞吐量约束。延迟注入采用双向对称模式，确保 $P_1$ 到 $P_2$ 与 $P_2$ 到 $P_1$ 的传输延迟一致。这种设计使得实验能够在受控条件下评估协议在不同网络场景中的性能表现。

## 与论文内容的对应关系

本图对应论文中实验设置部分的网络环境描述，为后续通信复杂度测量和端到端执行时间分析提供了基础设施说明。图中所示的拓扑结构支撑了Table 9至Table 11中报告的所有性能测量数据的采集工作。
