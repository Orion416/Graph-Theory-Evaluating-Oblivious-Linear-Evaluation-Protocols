# Figure 10: Communication Complexity Scaling 说明文档

## 图表概述

本图以双对数坐标系展示了五种OLE协议通信复杂度随问题规模的增长趋势。横轴为OLE实例数量 $N$，纵轴为通信量，两轴均采用对数刻度。每种协议用不同颜色、标记和线型区分，误差棒表示多次测量的置信区间。

## 复杂度理论基础

OLE协议的通信复杂度可归纳为三类渐近行为。线性复杂度 $O(N)$ 意味着通信量与实例数量成正比，IKNP-OLE和Noisy-Encoding OLE属于此类。亚线性复杂度 $O(\sqrt{N})$ 表示通信量增长慢于实例数量，PCG-OLE通过伪随机相关生成器实现了这一特性。准常数复杂度 $O(1)$ 表示通信量几乎不随规模增长，RLWE-OLE和HE-OLE借助环结构的打包能力接近这一行为。

图中灰色虚线标注了 $O(N)$ 和 $O(\sqrt{N})$ 两条参考曲线，便于对比各协议的实际复杂度与理论界限的吻合程度。

## 各协议表现分析

RLWE-OLE的通信曲线在低规模区域呈现常数特征，这源于环多项式能够打包 $n$ 个域元素于单个密文中。当 $N$ 超过环维度 $n$ 时，曲线开始呈现缓慢上升趋势。HE-OLE表现出类似特征，但由于同态加密密文开销较大，其绝对通信量高于RLWE-OLE。

IKNP-OLE呈现标准的线性增长模式，斜率接近1，这与其每OLE需要 $O(k)$ 次OT扩展的理论分析一致，其中 $k$ 为域元素位长。Noisy-Encoding OLE的斜率略高于IKNP-OLE，反映了多项式编码引入的额外开销。

PCG-OLE的曲线斜率约为0.5，验证了其亚线性通信复杂度的理论预期。在大规模场景下，PCG-OLE相比线性协议展现出通信效率优势。
